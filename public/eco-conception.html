Les principes

Ce que nous
faisons concrètement

Design sobre

Pas d'animation lourde, pas de carrousel, pas de fonctionnalité gadget. Chaque élément justifie sa présence.

Pages légères

HTML statique, un seul fichier CSS partagé, un seul fichier JS minimal. Pas de framework lourd.

Images optimisées

Dimensions adaptées à l'usage, lazy loading sur toutes les images hors écran, formats modernes recommandés (WebP/AVIF).

Polices limitées

Deux familles typographiques (Barlow Condensed + Inter), deux graisses chacune, chargées depuis Google Fonts avec display=swap.

Scripts limités

Un seul fichier JS de quelques lignes (navigation mobile). Aucun tracker, aucune bibliothèque externe.

Navigation claire

Architecture simple : 7 pages, navigation directe, formulaires adaptés par offre. Pas de redirection inutile.

Icônes SVG inline

Pictogrammes intégrés en SVG inline. Aucune librairie d'icônes externe chargée.

Accessibilité

Balises sémantiques, attributs ARIA, contrastes conformes WCAG AA, navigation clavier fonctionnelle.

Actions en cours

Amélioration
continue

L'éco-conception est un processus itératif. Voici les prochains points à traiter.

Convertir les images JPEG/PNG en WebP ou AVIF
Vérifier et documenter le score EcoIndex après déploiement
Documenter l'hébergement (localisation, énergie renouvelable)
Sous-setter les polices Google Fonts pour ne charger que les caractères utilisés
Ajouter un cache HTTP long sur les assets statiques
Mesurer le bilan carbone annuel du site
